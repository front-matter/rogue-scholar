Versionnage des DOI

Qu'est-ce que le versionnage des DOI ?

Le versionnage des DOI vous permet de :

  • modifier/mettre à jour les fichiers d'un enregistrement après leur publication.
  • citer une version spécifique d'un enregistrement.
  • citer toutes les versions d'un enregistrement.

Comment fonctionne le versionnage des DOI ?

Lors de la première publication d'un dépôt, deux DOI sont enregistrés :

  • un DOI représentant la version spécifique de votre enregistrement.
  • un DOI représentant toutes les versions de votre enregistrement.

Par la suite, un nouveau DOI est enregistré pour chaque nouvelle version de votre dépôt.

Un exemple concret est celui d'un logiciel. Si le logiciel a été publié en deux versions (v1.0 et v1.1), les DOI suivants auraient été enregistrés :

  • v1.0 (version spécifique) : 10.1234/7s0da-05p39
  • v1.1 (version spécifique) : 10.1234/chc5z-w0r30
  • Concept (toutes les versions) : 10.1234/3s67k-pd636

Les deux premiers DOI pour les versions v1.0 et v1.1 représentent les versions spécifiques du logiciel. Le dernier DOI représente toutes les versions du logiciel, c'est-à-dire le concept du logiciel et l'ensemble de ses versions. On les appelle donc respectivement DOI de version et DOI de concept (techniquement, ce sont tous des DOI normaux).

Vous remarquerez que les DOI de version ne comportent pas de suffixe « .v1 ». Lisez la suite pour savoir pourquoi.

Quel DOI dois-je utiliser dans les citations ?

Il est généralement recommandé d'utiliser le DOI de la version spécifique de votre enregistrement dans les citations, afin de garantir que d'autres chercheurs puissent accéder à l'exacte ressource scientifique que vous avez utilisée. Par défaut, nous utilisons la version spécifique pour générer les citations.

Vous pouvez utiliser le DOI de concept représentant toutes les versions dans les citations lorsqu'il est souhaitable de citer une ressource en évolution, sans préciser la version.

Vers où redirige le DOI de concept ?

Le DOI de concept redirige vers la page de destination de la dernière version de votre enregistrement.

Je souhaite uniquement modifier le titre de mon dépôt, vais-je quand même obtenir un nouveau DOI ?

Non, vous pouvez continuer à modifier les métadonnées de votre dépôt sans créer une nouvelle version de l'enregistrement. Vous ne devez créer une nouvelle version que si vous souhaitez mettre à jour les fichiers de votre enregistrement.

Pourquoi les DOI n'ont-ils pas de suffixe de numéro de version comme « .v1 » ?

Inclure des informations sémantiques telles que le numéro de version dans un DOI est une mauvaise pratique, car ces informations peuvent changer au fil du temps, alors que les DOI doivent rester persistants et ne pas changer.

De plus, le versionnage des DOI est linéaire, ce qui signifie que le numéro de version peut en fait ne pas correspondre au numéro de version réel de la ressource. Prenons par exemple un logiciel, où il est courant d'avoir des versions en points et de publier de nouvelles versions dans un ordre non linéaire (par ex. d'abord v1.0, puis v1.1, puis v2.0, puis v1.2).

Le suffixe de version n'est pas non plus une fonctionnalité du système DOI, c'est-à-dire qu'ajouter .v2 à un DOI ne redirigera pas vers la version 2 d'une ressource, quel que soit le fournisseur. Les différents fournisseurs utilisent également des modèles différents, tels que .v2, .2 ou /2.

Plus important encore, les suffixes de version ne sont pas lisibles par les machines. Un système de découverte comprenant les DOI ne saura pas que .v1 et .v2 d'un DOI sont en réalité deux versions de la même ressource.

Une meilleure solution consiste à lier sémantiquement deux DOI dans les métadonnées d'un DOI. Cela garantit que les systèmes de découverte disposent d'un moyen lisible par machine pour identifier que deux DOI sont des versions de la même ressource.

Voir également Cool DOIs pour plus d'informations.

Les fichiers sont-ils dupliqués pour chaque nouvelle version d'un enregistrement ?

Non. Si vous modifiez un fichier README de 10 ko dans un jeu de données de 50 Go, nous ne dupliquons pas l'intégralité des 50 Go. InvenioRDM, la plateforme logicielle sous-jacente, gère efficacement le stockage des fichiers et n'enregistre que les 10 ko supplémentaires.
